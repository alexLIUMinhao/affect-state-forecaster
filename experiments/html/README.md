# HTML 结果目录说明

这个目录用于集中查看实验的可视化报告页面。  
建议不要直接在根目录里逐个猜页面，而是先看 [index.html](./index.html) ，再按下面的类别进入。

## 推荐查看顺序

1. `index.html`
   总入口。先看这里，确认当前有哪些实验类别，以及每类应该看什么。

2. `main/`
   主实验目录。论文主表、当前核心结论、主要模型对比，优先看这里。

3. `data_analysis/`
   数据分析总览。适合先看数据本体分布、标签噪声、结构/时间统计，以及 7 类模型对数据的归纳性偏置。

4. `progress/`
   论文进展总览。适合快速看当前最优模型、证据状态、下一步计划。

5. `ratio_sweep/`、`cross_event/`、`seed_sweep/`
   这三个目录主要用于看稳健性和泛化，不直接替代主实验结论。

## 各目录含义

### `main/`

- 含义：主实验页面。
- 作用：用于论文主表和当前阶段的核心结果判断。
- 应该看什么：
  - 哪个模型当前最优
  - 新 baseline 是否进入主表候选
  - 主任务指标是否优于现有方法
- 当前最重要的页面通常在这里。

### `data_analysis/`

- 含义：数据分析与偏置解释页面。
- 作用：在继续堆实验之前，先回答“这份 benchmark 本身长什么样、哪里有风险、为什么某些方法更吃香”。
- 应该看什么：
  - 原始数据和 benchmark 是怎么构造出来的
  - 事件不均衡、`neutral` 占主导、弱标注噪声这些风险
  - 结构 / 时间 / 文本视角下的数据分布和低维可视化
  - `text / temporal / structure / affect-state / patchtst / timesnet / thread-transformer`
    这 7 类模型各自偏好利用什么规律
- 适合回答：
  - “当前结果是不是主要被数据分布驱动？”
  - “为什么结构模型或时序模型会更强/更弱？”
  - “下一步该优先补数据分析还是继续改模型？”

### `topconf/`

- 含义：顶会 baseline 的独立验证页面。
- 作用：只评估新增顶会方法是否值得纳入主表。
- 应该看什么：
  - `patchtst_baseline`
  - `timesnet_baseline`
  - `thread_transformer_baseline`
  的单独表现
- 注意：这里不是最终论文主结论，是否进入主表要回到 `main/` 看。

### `ratio_sweep/`

- 含义：不同 observation ratio 下的实验页面。
- 作用：验证模型在不同早期观测窗口下是否稳定。
- 应该看什么：
  - `30/70`
  - `50/50`
  - `70/30`
 这类设置下性能是否一致
- 适合回答：“模型是否只在某一个 ratio 上有效？”

### `cross_event/`

- 含义：跨事件泛化实验页面。
- 作用：测试模型在留一事件外设定下是否还能工作。
- 应该看什么：
  - 不同 held-out event 上的表现
  - 泛化是否明显下降
- 适合回答：“模型是否只记住了特定事件，而不是学到可迁移规律？”

### `seed_sweep/`

- 含义：随机种子稳定性实验页面。
- 作用：检查模型结果是否稳定、是否对初始化过于敏感。
- 应该看什么：
  - 不同 seed 下 MAE / RMSE 波动
  - 结论是否稳定
- 注意：这里主要看稳定性，不是主表排名依据。

### `progress/`

- 含义：论文进展总览页。
- 作用：给当前阶段做高层总结。
- 应该看什么：
  - 当前最优模型
  - 当前证据状态
  - 当前主要风险
  - 下一步实验计划
- 适合在继续跑实验前先看一遍，避免重复劳动。

### `diagnostics/`

- 含义：诊断、消融、机制解释类实验页面。
- 作用：解释“为什么有效/为什么无效”，而不是只看最终分数。
- 常见内容：
  - capacity matched
  - fusion diagnostic
  - labeler robustness
  - source gate validation
- 适合回答：“我们的方法为什么这样设计？”

### `imports/`

- 含义：历史导入页面。
- 作用：保存通过同步脚本导入的旧实验记录，便于追溯。
- 注意：
  - 这里主要是归档用途
  - 一般不作为第一查看入口

### `smoke/`

- 含义：冒烟测试页面。
- 作用：验证训练、评测、报告链路是否跑通。
- 注意：
  - 这里只说明流程可运行
  - 不能用来做论文效果结论

## 根目录文件说明

### `index.html`

- 这是 HTML 报告总入口。
- 以后优先从这里进入，而不是直接在目录里找单个页面。

### `README.md`

- 就是当前这份说明文档。
- 当目录继续扩展时，优先更新这里的说明口径。

## 一个简单原则

- 看主结论：去 `main/`
- 看数据本体和模型偏置：去 `data_analysis/`
- 看高层摘要：去 `progress/`
- 看稳健性：去 `ratio_sweep/` / `cross_event/` / `seed_sweep/`
- 看机制解释：去 `diagnostics/`
- 看历史归档：去 `imports/`
- 看是否跑通：去 `smoke/`
