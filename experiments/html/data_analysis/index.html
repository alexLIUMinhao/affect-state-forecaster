数据优先分析报告

这份报告先回答数据问题，再解释模型结果。核心目标不是继续堆模型，而是判断当前 benchmark 的统计结构、标签偏置、结构/时间分布，以及 7 类模型各自的归纳性偏置为什么会在这份数据上吃亏或受益。

原始数据 预处理与 Benchmark 数据统计 低维可视化 模型偏置

A. 原始数据形式

PHEME 原始 thread 的核心组成是 source tweet、reaction tweets 和 structure.json。即使本地没有完整 raw 目录，当前 repo 的处理链路仍然清楚地定义了期望输入和规范化结果。

期望原始布局

data/raw/pheme/<event_name>/<thread_id>/
├── source-tweets/*.json
├── reactions/*.json
└── structure.json

当前数据风险

  • 事件分布明显不均衡，头部事件主导总体统计。
  • future_majority_sentiment 明显偏向 neutral，分类任务容易被多数类主导。
  • 存在极小事件（如仅 2 个 thread 的事件），cross-event 评测的方差会较大。
  • benchmark 依赖弱标注词典，reply-level 标签噪声可能直接限制预测上限。

E. 7 类模型的归纳性偏置解释

这一部分不是重新汇报分数，而是解释不同模型真正“看到”的数据视角，以及在当前事件不均衡、neutral-heavy、弱标注构造的 benchmark 上，各自更容易受益或受损于什么偏置。

model_nameinput_viewignoresprefersbenefitsrisksfit_event_imbalancefit_neutral_heavyneed_long_threadsneed_clean_structurelabel_noise_sensitivity
text_baselinesource_text + observed replies concat_texts显式时间顺序、reply 级结构、局部传播轨迹词汇共现、全局语义、事件关键词当事件主题和负面词强耦合时容易受益；对短线程也不敏感容易把事件话题当作情绪代理，且难区分同词不同传播形态中中低低中
temporal_baselinereply 文本序列显式树拓扑、全局结构摘要局部顺序、early-to-late 的文本演化当早期回复顺序能反映情绪转折时受益线程很短或时间噪声大时，序列模式会不稳定中中中高低中高
structure_baselineconcat_text + 8维树统计细粒度时间过程、reply 级交互细节树深度、分叉、root children、leaf ratio 等全局拓扑摘要在事件不均衡、neutral-heavy 但传播形态仍有差异时更稳对局部情绪转折和单条关键回复不敏感高高中中高中低
affect_state_forecastersource + temporal + structure，经 affect-state bottleneck没有显式建模更细粒度的未来生成过程先压缩当前群体状态，再预测未来当当前观察窗口中已存在稳定的群体情绪轮廓时受益若弱标注噪声大，affect-state 可能成为 noisy summary高高中中高中高
patchtst_baseline固定时间桶统计序列单条 reply 文本细节、非桶内的细粒度结构差异局部重复模式、分段时序形状、桶间平滑变化当未来 negativity 可由早期统计轨迹概括时受益对非常短线程、强非平稳噪声和极端稀疏样本较敏感中高高中低中
timesnet_baseline桶化后的多变量时序特征reply 级文本与树中的离散交互多尺度局部卷积模式、周期和趋势混合当桶化统计中存在稳定尺度结构时受益极短线程和事件级分布漂移会削弱卷积式归纳偏置中中高中低中
thread_transformer_baselinesource token + reply tokens + 结构 side features没有显式全局树统计先验reply 级局部交互、位置与结构偏置、token 细节当关键回复和局部结构模式决定未来时受益样本少、线程短、标签偏斜时更容易过拟合 token 级噪声中低中高高高
